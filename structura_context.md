## Mission

Structura exists to make AI automation accessible and practical for professionals and small-to-medium businesses (SMBs) in Colombia. The company demystifies artificial intelligence — a topic many clients perceive as complex or intimidating — and shows concretely how it can eliminate repetitive work, increase productivity, and free up time for the activities that truly drive value in a business or career.

---

## What Structura Does

Structura provides end-to-end AI automation consulting structured in three service pillars:

### 1. Diagnosis — Identify what can be automated
- Conduct structured interviews and working sessions with the client's team
- Map existing workflows (administrative, operational, teaching, communications, etc.)
- Identify repetitive, time-consuming tasks that are candidates for AI automation
- Produce a prioritized automation roadmap

### 2. Design — Build the automation tools
- Design and develop AI-powered tools tailored to the client's workflows
- Tools are built using the Structura core stack (see below)
- Automation is explained in plain, non-technical language accessible to any team member
- Solutions are practical, not theoretical — every tool must go live and be used

### 3. Deployment & Training — Implement and transfer knowledge
- Install and configure tools in the client's environment
- Conduct training sessions so staff can operate tools independently
- Provide go-live support and iteration based on real-world feedback

## Technology Stack

Structura uses a defined, opinionated tool stack to deliver consistent, high-quality automations:

| Tool | Role |
|---|---|
| **Claude (Anthropic)** | Core AI engine — Projects for institutional context, Skills for standalone tasks, Artifacts for deliverables |
| **Claude Cowork** | Desktop automation for non-developers; file and task management |
| **NotebookLM** | Knowledge base and document Q&A for teaching/research workflows |
| **Google Sheets / Excel** | Data management, reporting, structured outputs |
| **n8n** | Advanced workflow automation and system integrations (Phase 3 engagements) |

### Tool Selection Philosophy
- **Claude Projects vs. Claude Skills:** Projects are used for context-rich, ongoing institutional workflows (e.g., a school's communication processes). Skills are used for standalone, repeatable quick tasks.
- **NotebookLM vs. Claude Projects:** NotebookLM is preferred when the primary need is querying and summarizing large document repositories. Claude Projects are preferred when the workflow requires generation, drafting, and multi-step reasoning.
- **Claude Cowork vs. n8n:** Cowork is introduced first for non-technical users managing files and local tasks. n8n is reserved for advanced phases where cross-system integrations are needed.

## Delivery Model

Engagements are structured in phases, typically:

- **Phase 1 (months 1–4):** Administrative automation — forms, scheduling, reports, communications
- **Phase 2 (months 3–7):** Teaching / knowledge workflow automation — document Q&A, lesson prep, content generation
- **Phase 3 (months 7–12):** Advanced AI integration — cross-system workflows, n8n pipelines, institutional AI infrastructure

Each phase follows a repeatable delivery cycle:
1. Staff interviews and working sessions
2. Automation development
3. Software installation and configuration
4. Training session
5. Go-live and iteration
